## Acceptance Criteria
*(verbatim from `docs/plan.md:461-468` — do not rephrase. AC-9 / AC-10 NEW per A-17 / A-18 amendments below.)*

- **AC-1 (one-command build):** *Given* a fresh clone, *when* `scons addon` runs, *then* `addons/SolanaSDK/` is produced with GDExtension binaries + AAR + GDScript + docs in one invocation; existing `SConstruct addon` alias is extended/validated; README has a 5-line "Build the addon" section.
- **AC-2 (PR-blocking CI):** *Given* every PR to `master`, *when* CI runs, *then* these jobs run and all must pass before merge: `build.yml` (Android + multi-platform GDExtension — existing), `clang_tools.yml` (existing), `kotlin_lint.yml` (new: ktlint + Android Lint), `kotlin_unit.yml` (new: `./gradlew test`), `cpp_host_unit_tests.yml` (new: GoogleTest on Linux/Windows/macOS), `gdextension_load_multi_platform.yml` (reuse of existing 4-platform matrix, extended to assert UNSUPPORTED_PLATFORM), `instrumented_emulator_fakewallet.yml` (new: macos-latest + emulator + Fake Wallet APK), `codegen_drift_check.yml` (new), `generator_tests.yml` (new: PR-blocking), `manifest_merge_audit.yml` (new), `r8_release_smoke.yml` (new), `reproducible_build.yml` (new: 2-runner byte-diff), `fixture_schema_validation.yml` (new), `header_lint.yml` (new: bans STL types in public C++ headers), `since_tags_check.yml` (new).
- **AC-3 (weekly cron):** *Given* `verify_parity.yml` cron, *when* it runs weekly Sunday 03:00 UTC, *then* it diffs the pinned RN SDK npm package against `docs/rn-parity.md` method list; on new method found, opens an issue tagged `parity-drift`; PR-blocking only during new-method follow-up PR.
- **AC-4 (CI SLO monitor):** *Given* the last 30 PRs, *when* `ci_slo_audit` (monthly cron) runs, *then* it reports P95 wall-clock, T3 flake rate, and number of retries; emits a dashboard comment on the tracking issue if P95 > 35 min or T3 flake > 2%.
- **AC-5 (retry policy):** *Given* T3 instrumented_emulator_fakewallet fails due to emulator boot failure (detected via log pattern `emulator boot timed out`), *when* the job runs, *then* it auto-retries exactly once; no retries on T1/T2/T5.
- **AC-6 (grep bans):** *Given* CI greps run, *then* the build fails if any of these patterns appear: (a) `WebSocket` or `ECDH` in hand-rolled form under `android/plugin/`, (b) `Log\.(d|v|i|w|e)\(` with `authToken|publicKey|SecretString.reveal` args, (c) `SdkLog\.(v|d|i|w|e)\([^{]*\$\{` (string interpolation without lambda), (d) any wallet package name string literal in a conditional (e.g., `if (wallet == "com.phantom.app")`), (e) `#ifdef __ANDROID__` outside `src/mwa/platform_selector.cpp`.
- **AC-7 (@since check):** *Given* `scripts/check_since_tags.py` runs on every PR, *when* a new public method/signal is added without an `@since` XML tag, *then* the build fails with the offending line.
- **AC-8 (compat-matrix enforcement):** *Given* CI validates `docs/compatibility-matrix.md`, *when* `build.gradle.kts` or `.gdextension` drifts from the matrix values (compileSdk, minSdk, clientlib-ktx version, Godot minor), *then* the build fails with a named drift.
- **AC-9 (NEW per A-17 / CR-5-4-F MANDATED — release-on-tag CI):** *Given* a git tag matching `v*.*.*` (or `v*.*.*-rc*`) is pushed, *when* `release_on_tag.yml` runs, *then* it builds multi-platform native libraries + AAR + GDScript + docs (matching the `build.yml` matrix for native targets), bundles `addons/SolanaSDK/` as `addons-bundle.zip`, and publishes a GitHub Release tied to the git tag with the ZIP attached + auto-generated changelog notes.
- **AC-10 (NEW per A-18 — OSI canonical MIT byte-diff PR-gate):** *Given* `tools/check_license_diff.sh` (shipped Story 5-5 T7 with embedded OSI canonical MIT body), *when* CI runs on every PR, *then* the script exits 0 against repo-root `LICENSE` (body byte-identical to OSI canonical MIT modulo the copyright-holder line); divergence fails the build with the offending diff.

## Architecture Guardrails

8 LOCKED design decisions covering: workflow shape (standalone vs folded), release-on-tag matrix scope, ci_slo_audit data source, since_tags_check parser approach, verify_parity upstream-drift gap (CR-5-5-C), grep_bans extension pattern (d) regex shape, retry-once detection mechanism, compat_matrix_drift source-of-truth parsing.

- **DD-5-6-1 (LOCKED — `byte_diff_license` packaging; Q3=(a) resolved 2026-04-30):** Standalone `byte_diff_license.yml` workflow per Q3 user direction (a). The workflow has one job: checkout → invoke `bash tools/check_license_diff.sh`. Trigger: `pull_request` + `push` to `master` + `feature/**` (mirroring the `grep_bans.yml` trigger shape). Cost: ~30 sec wallclock per PR. (Pivot path to (b) folded into release_on_tag.yml is now closed; T10 lands the standalone workflow.)
  - **Rejected alternative:** fold into `verify_docs.yml` — verify_docs is doxygen + markdown-link-check; LICENSE is neither. Mismatched workflow concern.

- **DD-5-6-2 (LOCKED — `release_on_tag.yml` matrix scope):** The release matrix builds the same 6 native targets `build.yml` covers today (linux-debug + linux-release + windows-release + macos-release + android-release-arm64 + android-release-x86_64 + web-release) PLUS the AAR via `./gradlew :plugin:assembleRelease`. Web is INCLUDED per current build.yml matrix even though `gdscript_tests.yml` web entry is symbol-resolve-only (DD-5-1-2) — release artifacts ship the .wasm so consumers building Web exports have a usable native lib. Bundle layout: `addons/SolanaSDK/` is the canonical staging dir produced by `scons addon`; the workflow zips this dir as `addons-bundle.zip` after each native target deposits its `.so` / `.dll` / `.dylib` / `.wasm` / `WalletAdapterAndroid-release.aar` into the staging dir per the existing `SConstruct:497-528` `Copy` actions. **GDScript + docs are auto-included** because they live under `addons/SolanaSDK/mwa/` and `addons/SolanaSDK/Dialogs/` etc. already on master.

- **DD-5-6-3 (LOCKED — `ci_slo_audit.yml` data source = GitHub API, NOT local cache):** The monthly cron queries `gh api repos/<owner>/<repo>/actions/runs?event=pull_request&per_page=100` for the last 30 PR-event runs, computes wall-clock per run from `created_at`/`updated_at`, identifies T3 (`instrumented_tests.yml`) runs explicitly by workflow name, counts `conclusion=failure` followed by `conclusion=success` (= retry pattern). Output: a markdown table posted as a comment on a designated tracking issue (`docs/concerns.md` references `CI SLO tracking issue #N`; T6 must wire the issue number — Q5 below if user wants this surfaced).
  - **Rejected alternative:** parse a local `state.json` cache — would require every workflow to write its wallclock to state.json (high coupling, not in scope for v0.1.0).
  - **Rejected alternative:** use a 3rd-party SLO service (e.g., Datadog) — out of grant scope; no infrastructure budget.

- **DD-5-6-4 (LOCKED — `scripts/check_since_tags.py` parser approach):** Python script using regex (NOT a full XML parser) — the XML doc-comment scope is small (only public-API-surface `.gd` and `.kt` files); a regex over `class_name`, `signal `, `func ` (GDScript) + `@UsedByGodot fun ` (Kotlin) + the immediately-preceding doc-comment block is sufficient and avoids a heavy XML dep. Catches: signal/method declared without `@since v<X.Y.Z>` in the preceding 5-line doc-comment block. Pytest cases (~6-10): (1) GDScript signal with `@since` → PASS; (2) GDScript signal without `@since` → FAIL; (3) Kotlin `@UsedByGodot fun` with `@since` → PASS; (4) Kotlin `@UsedByGodot fun` without `@since` → FAIL; (5) GDScript private `_fn` (no `@since` required) → PASS; (6) Empty file → PASS; (7-10) regression cases from existing source.
  - **Rejected alternative:** use Doxygen output — Doxygen is XML-based + slow + already wired in `verify_docs.yml`; a separate fast-running script is better suited for PR-time gates.
  - **Rejected alternative:** require `@since` on every public symbol retroactively — would require a one-time backfill of every existing public method/signal across the entire codebase. Out of scope. The script ONLY checks NEW additions (diff-aware via `git diff origin/master...HEAD` on `.gd` + `.kt` files).

  - **Rejected alternative:** mandatory PR-blocking from day 1 — a transient upstream registry outage would block PRs unrelated to parity. Informational-first → blocking-after-validation is safer.
  - **Rejected alternative:** skip upstream-fetch entirely (CR-5-5-C option (c)) — sustainable for v0.1.0 but adds zero detection capability beyond what 5-5 already shipped. T5 implements the upstream-fetch step gated on a `parity-fetch-upstream: 'true'` workflow input that the cron sets but PRs can override.

- **DD-5-6-6 (LOCKED — `grep_bans.yml` extension scope):** Today `ci/grep_bans.sh` patterns 1-7 cover AC-6 (b) (patterns 1+2+3 = Log.* with authToken/publicKey/reveal), AC-6 (c) (pattern 4 = SdkLog interpolation), AC-6 (e) (pattern 5 = `#ifdef __ANDROID__` outside `platform_selector.cpp`), plus DD-22 thread-marshalling (pattern 6) and `MWA_TESTING` define (pattern 7), plus signing-story patterns 8-10. **AC-6 (a) `WebSocket`/`ECDH` partial:** patterns may exist transitively via clientlib-ktx imports — narrow regex must scope to `android/plugin/src/main/` and exclude string literals citing the spec/protocol name. **AC-6 (d) wallet-conditional:** NEW pattern for T8 — regex matches `if\s*\(\s*\w+\s*==\s*"com\.[a-zA-Z0-9._]*(phantom|solflare|backpack|jupiter|exodus|trust|coinbase|ledger|brave|coin98)"` (literal-string equality against well-known wallet package names); ANY match in `android/plugin/src/main/` OR `addons/SolanaSDK/mwa/` is a violation.
