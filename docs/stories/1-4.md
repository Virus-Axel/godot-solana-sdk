# Story 1-4: MwaAndroidBridge Interface + Platform Selector TU + MockMwaAndroidBridge + GodotMainDispatcher

## User Story
As an SDK contributor, I want a platform-agnostic C++ bridge that returns `UNSUPPORTED_PLATFORM` on non-Android and delegates to JNI on Android, and a main-thread dispatcher that guarantees all signals cross to Godot main, so that non-Android builds don't need `#ifdef __ANDROID__` sprawl and signal marshalling is verifiable.

## Acceptance Criteria
*(verbatim from `docs/plan.md` §Story 1-4 — do not rephrase; implementation-detail clarifications belong in Architecture Guardrails / Dev Notes)*

- **AC-1 (interface):** *Given* `src/mwa/include/mwa_android_bridge.hpp` declares `class MwaAndroidBridge` with pure-virtual methods for each of the 7 MWA ops + lifecycle, *when* compiled, *then* all public headers under `src/mwa/include/` reference only `godot::` types (no `std::`); verified by CI header-lint.
- **AC-2 (platform selector):** *Given* `src/mwa/platform_selector.cpp` is the only conditional-compilation TU, *when* built for non-Android, *then* `MwaAndroidBridge::create()` returns a no-op impl that emits `mwa_error{code=UNSUPPORTED_PLATFORM, message="MWA is Android-only; OS.get_name()=<name>"}` within 1 frame for every method; *when* built for Android, returns `MwaAndroidBridgeJni` stub (no JNI calls yet — just logs and returns `NOT_CONNECTED`).
- **AC-3 (MockMwaAndroidBridge):** *Given* `MockMwaAndroidBridge` test double, *when* a host-platform unit test (`MWA.is_supported == false` on Linux etc.) invokes a method, *then* it records the call + args + lets the test drive the callback on a named thread.
- **AC-4 (GodotMainDispatcher):** *Given* `class GodotMainDispatcher` wraps a `Ref<MobileWalletAdapter>` node and exposes `post(signal_name: String, payload: Dictionary)`, *when* invoked from any thread, *then* the emission runs on Godot main thread via `call_deferred("emit_signal", signal_name, payload)`.
- **AC-5 (architectural check):** *Given* CI greps source, *when* any C++ TU outside `platform_selector.cpp` contains `#ifdef __ANDROID__`, *then* the build fails with a named file and line.
- **AC-6 (error case):** *Given* `GodotMainDispatcher` called before the node is registered (e.g., during engine teardown), *when* `post()` runs, *then* the call is silently dropped with `SdkLog.w("dispatcher post after destroy")` — no crash.

## Architecture Guardrails

### DD-12 [LOCKED]: MwaAndroidBridge C++ interface + platform selector
`MwaAndroidBridge` is a pure-abstract C++ class (NOT a Godot-registered class). `MwaAndroidBridgeJni` is its Android-only concrete impl; a sibling no-op impl handles every non-Android platform. A single translation unit (`src/mwa/platform_selector.cpp`) is the ONLY source allowed to `#ifdef __ANDROID__` — it selects and returns the appropriate impl from `MwaAndroidBridge::create(...)`. This prevents `#ifdef __ANDROID__` sprawl across the codebase and enables `MockMwaAndroidBridge` for host-platform tests.

### DD-22 [LOCKED]: Single thread-marshalling rule
Every signal to GDScript crosses to the Godot main thread via `call_deferred("emit_signal", ...)` in the C++ `GodotMainDispatcher`. Kotlin NEVER emits signals directly to GDScript. All Kotlin → GDScript signal paths route through the dispatcher. An architectural grep test enforces (1) Kotlin has no direct `emitSignal` to the host Godot plugin, (2) C++ has no `emit_signal` outside `GodotMainDispatcher`.

### DD-26 [LOCKED]: ABI hygiene — `godot::` types only in public headers
Public C++ headers under `src/mwa/include/` use ONLY `godot::` types (`godot::Variant`, `godot::String`, `godot::Dictionary`, `godot::PackedByteArray`, `godot::Ref<T>`, `godot::Array`, `godot::TypedArray<>`). No `std::string`, `std::vector`, `std::unique_ptr`, `std::function`, or other STL templates. Banned `#include`s in `src/mwa/include/*.hpp`: `<string>`, `<vector>`, `<memory>`, `<functional>`, `<iosfwd>`, `<ostream>`, `<iostream>`, `<sstream>`, `<map>`, `<set>`, `<unordered_map>`, `<unordered_set>`, `<deque>`, `<list>`, `<thread>`, `<mutex>`, `<future>`, `<chrono>`. **ALLOWED** in public headers: C-compat fixed-width integer headers `<cstdint>` (for `int32_t`, `uint64_t`) and `<cstddef>` (for `size_t`) — these are ABI-stable across libstdc++/libc++/MSVC and do not leak STL container types. Impl `.cpp` files (NOT under `src/mwa/include/`) may use `std::` freely — the lint scan only covers the public header boundary. CI header-lint (AC-1, Task 7) scans `src/mwa/include/` and fails on any banned identifier or include.

### `create()` signature resolution
Plan.md AC-2 prose reads `MwaAndroidBridge::create()` (elided arg list). The dispatchers must be injected into both impls (no-op + JNI stub) so they can route `UNSUPPORTED_PLATFORM` / `NOT_CONNECTED` emissions. **Authoritative signature for Story 1-4:**

```cpp
namespace godot_solana_sdk::mwa {
class GodotMainDispatcher;  // forward-decl
class MwaAndroidBridge {
public:
    virtual ~MwaAndroidBridge() = default;
    static MwaAndroidBridge* create(GodotMainDispatcher* dispatcher);
    // 7 ops + lifecycle pure-virtual methods here
};
}
```

Caller (Story 1-5's `MobileWalletAdapter` node) constructs the dispatcher first, then passes it to `create(...)`. Bridge impls store a raw pointer to the dispatcher (no ownership — dispatcher outlives the bridge because both die together during engine teardown, and `get_instance_from_id` handles the null-case if teardown reorders them).

### 7 MWA ops + lifecycle surface (pure-virtual on `MwaAndroidBridge`)
Operations (full set, consumer-driven by Story 1-5+): `connect`, `reauthorize`, `disconnect`, `deauthorize`, `sign_messages`, `sign_transactions`, `sign_and_send`, plus lifecycle (`forget_all`, `get_diagnostics`). Each method takes a `godot::String request_id` for correlation and returns `void` — results arrive asynchronously via `GodotMainDispatcher::post()` emitting `mwa_error` / `mwa_timeout` / `{op}_completed` signals.

### `mwa_error` signal payload (from `docs/architecture.md` §3.2)
```gdscript
{
  "request_id": String, "code": String, "user_message": String,
  "developer_details": String, "recoverable": bool,
  "retry_hint": String,  # "retry_now" | "reauthorize" | "install_wallet" | "connect" | "none" (A-1)
  "layer": String, "cause": String, "source_method": String,
}
```

### Signal-emit discipline (DD-22)
No C++ TU outside `src/mwa/godot_main_dispatcher.cpp` may call `emit_signal` directly. Grep-enforced at CI (AC-5 extension — see Tasks §T6).

**"Within 1 frame" (AC-2) interpretation:** `dispatcher->post(...)` is called synchronously on the bridge method's calling thread; the `call_deferred` inside `post()` enqueues the emit for the target's next idle sync point. If the caller is already on main thread, the emit lands this frame's idle phase (the frame of the call). If the caller is on a worker thread, the emit lands at the next main-thread idle sync. Either way, the signal fires before the second `_process` tick of the target node. Story 1-5's thread-hop test asserts this bound with a 16ms timeout.

**`GDCLASS` / `GDCLASS_CUSTOM` (A-11):** No macros on any class in this story.
