## Architecture Guardrails

### §2.2 Component H — SecureTokenStore [LOCKED]
> Ref: `docs/architecture.md#8.3 Secure Storage Implementation` — inlines the full Kotlin class shell (MasterKey builder, EncryptedSharedPreferences.create, wipeCorruptPrefs, MASTER_KEY_ALIAS = "godot-sdk-mwa-master-key-v1", PREFS_FILE_NAME = "godot-sdk-mwa-tokens-v1"). Use the inlined shell verbatim; do not invent alternate aliases.

**CacheKey** (inlined, load-bearing — AC-1/AC-2):
```kotlin
data class CacheKey(
    val cluster: String,
    val chainId: String,
    val walletPackage: String,
    val identityUri: String,
) {
    /** `"mwa::v1::" + SHA256(canonical())` (DD-17 + DD-27). */
    fun toHash(): String = "mwa::v1::" + sha256Hex(canonical())

    /** Length-prefixed concatenation: `len:4-big-endian || bytes` for each field, concatenated.
     *  Prevents delimiter collision (DD-27). A raw `|` joiner would collide on
     *  `identityUri` containing a literal `|` in a URL query (AC-2). */
    private fun canonical(): ByteArray = ByteArrayOutputStream().use { out ->
        listOf(cluster, chainId, walletPackage, identityUri).forEach { s ->
            val b = s.toByteArray(Charsets.UTF_8)
            out.write(ByteBuffer.allocate(4).order(ByteOrder.BIG_ENDIAN).putInt(b.size).array())
            out.write(b)
        }
        out.toByteArray()
    }
}
```
Do NOT use `"|".join(...)` — AC-2 explicitly calls out the delimiter-collision attack surface. Repo convention for hex encoding: lowercase, no separator (`%02x` per byte).

### §7.3 Watchdog + terminal-signal invariant [LOCKED] — DD-23
> Ref: `docs/architecture.md#7.3 Watchdog & Terminal-Signal Invariant` — rationale.

- Effective watchdog = `min(internal_default=60000, opts.timeout_ms=120000)` ms. Single `kotlinx.coroutines.withTimeoutOrNull(effective)` wraps the `MwaClientImpl.authorize` coroutine; NO nested timeouts.
- On expiry: cancel coroutine, emit `mwa_timeout{request_id, source_method:"connect", elapsed_ms, watchdog_ms}`, mark `request_id` as TERMINATED in an in-memory `InflightMap`.
- **Terminal-signal invariant** (AC-6 load-bearing): for any `request_id`, AT MOST ONE terminal signal ever fires across `{connect_completed, mwa_error, mwa_timeout, mwa_cancelled_lifecycle}`. Late results arriving after a terminal emission are: (1) logged via `SdkLog.w { "late_result corrId=$corrId" }`, (2) increment `MwaDiagnostics.lateResultCount`, (3) dropped. Tracked per-op via an in-memory `ConcurrentHashMap<String, InflightState>` in the Kotlin plugin layer.

### D-5 [LOCKED] — CR-18 resolution: production uses `godot::Callable`; `target_id_` moves under `#ifdef MWA_TESTING`
**Rejected option 1 (`_exit_tree` destroying-flag + join-in-flight):** does NOT close the ABA window (flag is per-target; a reused ObjectID slot is a different target, CR-18 Extension 2026-04-23). Also requires Kotlin-side cancellation signaling for in-flight clientlib coroutines, bleeding teardown concerns into the bridge impl.

**Chosen option 2 (`godot::Callable` carrying target+method handle, with `target_id_` moved under MWA_TESTING for the retained sync-drain harness):** The CR-18 hazard is the production `#else` branch that does `ObjectDB::get_instance(target_id_) → raw-ptr virtual dispatch`. We eliminate it by binding a `godot::Callable emit_signal_callable_` at ctor and replacing the production branch with a single `emit_signal_callable_.call_deferred(...)` call. Callable's resolution is atomic + lifetime-aware (godot-cpp `Callable` holds an ObjectID under the hood but the call machinery performs a single atomic resolve-then-dispatch; a freed target produces a no-op with an internal warning, not a UAF).

**But we CANNOT fully delete `target_id_`** — A-13's preservation clause explicitly keeps `drain_for_testing` + `pending_` + `snapshot_pending_for_testing` in source for CR-35's future headless-Godot tier. `drain_for_testing` is a SYNCHRONOUS dispatch harness (test code calls it after worker threads have joined, expects `emit_signal` to fire before return); `Callable::call_deferred` is asynchronous (posts to the message queue, returns immediately). The two contracts are incompatible by design. Therefore:

```cpp
// godot_main_dispatcher.hpp (new production contract)
private:
    godot::Callable emit_signal_callable_;   // production field — replaces target_id_ in the hot path

#ifdef MWA_TESTING
    godot::ObjectID target_id_;               // MWA_TESTING-only — kept for drain_for_testing() sync dispatch via ObjectDB::get_instance
    godot::Array pending_;
    mutable godot::Mutex drain_mutex_;
#endif
```

Ctor wires both under MWA_TESTING, only `emit_signal_callable_` in production:
```cpp
GodotMainDispatcher::GodotMainDispatcher(godot::Object* target) {
    ERR_FAIL_NULL_MSG(target, "...");
    emit_signal_callable_ = godot::Callable(target, "emit_signal");
#ifdef MWA_TESTING
    target_id_ = target->get_instance_id();
#endif
}
```

- Production `post()` (`#else` branch, new): `emit_signal_callable_.call_deferred(signal_name, <arity-ladder-unpacked args>)`. See D-6 for the arity ladder.
- MWA_TESTING `post()` (enqueue branch, unchanged shape — args payload key renamed per D-6): append `{signal_name, args}` to `pending_` under `drain_mutex_`, return.
- MWA_TESTING `drain_for_testing()`: snapshot+clear under lock; outside the lock, for each entry, resolve via `godot::ObjectDB::get_instance(target_id_)` (NOT via `emit_signal_callable_`), and call `target->emit_signal(signal_name, <unpacked args>)` — synchronous, matches the retired harness contract. Uses its own small arity ladder identical to production.

Net effect: CR-18 is closed in the production branch (no raw-pointer virtual dispatch); MWA_TESTING branch still works as the 1-5-built harness intends; A-13's "retain source" clause is honored in substance, not just in syntax.

### D-6 [LOCKED] — CR-28 resolution: dispatcher `post` takes `godot::Array` payload + HARD-CODED ARITY LADDER (bounded `{1, 2}`)
**Rejected option a (variadic template):** template in a public header drags template-instantiation hygiene across DD-26's ABI-boundary (`src/mwa/include/godot_main_dispatcher.hpp`). Template signatures are not stable across compiler versions — DD-26's whole point is stable engine-types-only ABI.

**Rejected option c (overload pair):** two `post()` overloads (1-Dict + 2-arg) proliferates as the `*_completed` family grows. Migration-unsafe.

**Rejected option d (runtime-length argv through `Callable::callv` / `call_deferredv`):** godot-cpp's extension_api.json (verified 2026-04-23) exposes Callable as: `call(VARARGS)`, `call_deferred(VARARGS)`, `callv(arguments: Array)`. `callv` is **synchronous** — does not post to the message queue. There is **no `call_deferredv(Array)` form** and no public `Callable::callp_deferred(const Variant**, int)` surface. Attempting to "build a `const Variant**` argv and hand it to `call_deferred`" conflates the sync array form with the deferred variadic template — it does not compile.

**Chosen option b (`godot::Array` payload, 1-param API + hard-coded arity ladder):**
```cpp
void post(const godot::String& signal_name, const godot::Array& args);
```
- For the 4 error/lifecycle signals (`mwa_error`, `mwa_timeout`, `mwa_cancelled_lifecycle`, `reauth_required`) — 1-elem array: `[payload_dict]`.
- For the 7 `*_completed` signals (per A-12) — 2-elem array: `[request_id_string, result_dict]`.
- **Arity upper bound:** 2, bounded by A-12. MWA ships no 3+ param signals and will not in Epic 2-5 per the current spec (every new signal introduced post-2-1 still belongs to one of the two families). If a future story needs arity 3, it MUST file a new amendment AND extend the ladder.
- Dispatcher unpacks via a hard-coded arity ladder (production `#else` branch):
  ```cpp
  // src/mwa/godot_main_dispatcher.cpp — production branch (post-T6)
  const int n = args.size();
  switch (n) {
      case 1: emit_signal_callable_.call_deferred(signal_name, args[0]); break;
      case 2: emit_signal_callable_.call_deferred(signal_name, args[0], args[1]); break;
      default:
          ERR_FAIL_MSG(godot::vformat(
              "GodotMainDispatcher::post: unsupported arity %d for signal '%s' (expected 1 or 2)",
              n, signal_name));
  }
  ```
  The `default` arm is not a silent drop — it is a loud developer-facing failure via `ERR_FAIL_MSG` so an out-of-contract call is impossible to ship.
- MWA_TESTING `drain_for_testing()` uses an identical ladder against the post-resolved `target->emit_signal(...)` path (see D-5). The two ladders are symmetrical — if one grows, the other MUST grow in the same commit.
- **T6 pre-work verification (locked in now, not discovered at implementation time):** before writing the ladder, implementer confirms extension_api.json at `godot-cpp/gdextension/extension_api.json` lists `call_deferred` with `is_vararg: true` and `callv` with an `Array` argument but NO `is_vararg`. If godot-cpp exposes a newer `call_deferredv(Array)` entry in a minor release, this D-6 ladder can be retired — but that is a future-story decision, not a 2-1 scope expansion.
- **Callers affected:** `NoOpMwaAndroidBridge` (1-4), `MwaAndroidBridgeJni` (2-1 T5), `MobileWalletAdapter::mwa_connect` null-bridge branch (1-5 T3 → rewritten in T6). All migrate from `dispatcher_->post(signal, payload_dict)` to `dispatcher_->post(signal, godot::Array::make(payload_dict))` for 1-param signals or `godot::Array::make(request_id, result)` for 2-param signals.

## Risks

1. **CR-18 resolution surfaces a new bug.** Callable's behavior under node-destroy-during-deferred-dispatch is not as battle-tested as raw `ObjectDB::get_instance` + null-check. Mitigation: T9's AC8 logcat scan doubles as a crash-free smoke; if Callable fails during teardown, the test will either crash (caught in CI) or leave a `push_warning` residue (inspection-only). If surfaced, fall back to option 1 (`_exit_tree` barrier) in a follow-up story — do NOT paper over in 2-1. Escalate as a concern.
2. **Dispatcher API break disturbs downstream stories.** Every caller of `dispatcher_->post(...)` in the tree migrates in T6. Stories 2-2, 2-3, 3-x, 4-x inherit the new `godot::Array` signature; plan.md's inline examples are behind the rename. Mitigation: T6 landing must migrate ALL known call sites in one commit; no half-migrated state. If a downstream story was drafted against the old 1-Dict signature, update during story-creation at that time.
3. **`JavaVM*` caching race at JNI_OnLoad.** Multiple AAR loads (uncommon but possible under test-runner config) could race on `g_jvm`. Mitigation: `static std::once_flag` + `std::call_once` in `JNI_OnLoad`; or use `std::atomic<JavaVM*>` CAS. Standard JNI idiom; not an MWA-specific hazard.
4. **Emulator flakes in T9 androidTest.** Per DD-20, macos-latest + android-emulator-runner is the supported harness but historically flakes at ~5% rate. Mitigation: retry-once on `connectedAndroidTest` step; if retry still fails, file a flaky baseline entry (per retrospect/test-integrity) and re-present at 5-story checkpoint. Do NOT auto-mark as pre-existing.
5. **Fake Wallet APK drift.** If `testdata/fake-wallet-debug.apk` is not checked in (or is out of date), T9 can't run. Mitigation: T9's CI workflow includes a build step compiling Fake Wallet from source if APK missing. Source path convention: `testdata/fake-wallet/` (alongside the fixtures dir). Check with T0 pre-implementation.
6. **Robolectric shim for `EncryptedSharedPreferences` incomplete.** Robolectric 4.11+ has partial shims but not all MasterKey paths are mocked. Mitigation: T8 uses the emulator for any AC that crosses the MasterKey boundary (AC-1 plaintext-scan, AC-7 fingerprint) — those move to T9. Robolectric-viable Kotlin unit tests are AC-4 (classification), AC-5 (payload shape), AC-6 (timeout + late-drop logic) + the pure-helper T2/T3 tests.
7. **A-14 retrofit affects Story 1-4's NoOp bridge + Story 1-5's null-bridge branch consumers.** Neither code path changes (both already emit the 10-key shape); the amendment simply blesses the existing shape as canonical. No backward-incompat risk.
8. **CR-32 GDScript test requires a headless Godot binary on the Linux runner.** Godot 4.3+ supports `--headless --script <test.gd>`. Mitigation: T10 wires `softprops/action-gh-release` or equivalent to download the Godot binary pinned at 4.3.0-stable. If the download step is slow (~80 MB), cache via `actions/cache` keyed on Godot version.
9. **Dependency on CR-34 mitigation being option 2 not option 1.** T6's D-5 resolution is option 2 (Callable). If at implementation time the implementer discovers that godot-cpp 4.3.0 + clang on Android has a blocking bug in `Callable::call_deferred`, the fallback is option 1 (`_exit_tree` barrier) — NOT a partial mix. Escalate as Rule-2 deviation with amendment if the switch is needed.
10. **`@VisibleForTesting` ctor-overload for `GDExtensionAndroidPlugin` may clash with Godot plugin loading semantics.** Godot instantiates the plugin via reflection looking for a `(Godot)` ctor. Mitigation: the test ctor takes `(Godot, MwaClient)` — Godot's reflection picks the 1-arg ctor; tests pick the 2-arg ctor. If reflection is ambiguous, split into a factory pattern like `MwaClientImpl`'s `adapterFactory`.
