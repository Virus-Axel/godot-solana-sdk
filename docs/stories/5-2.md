# Story 5-2: get_diagnostics() + Correlation Ring Buffer + HKDF Fingerprint + device_posture + get_suggested_wallet_install_links

## User Story
*(verbatim from `docs/plan.md:373` — do not rephrase)*

As a support engineer triaging a user's "signal never fired" bug report, I want `MWA.get_diagnostics()` to return the last 20 correlation IDs, session state, versions, and a non-sensitive auth_token fingerprint, so that I can match logcat traces to user-reported symptoms without needing the device.

## Architecture Guardrails

5 LOCKED design decisions covering: SYNC/ASYNC seam reshape, correlation-trace recorder cross-cut centralization, non-Android empty-Dict semantics, AC-5 key-shape reconciliation, CR-3-3-A closure scope.

- **DD-5-2-1 (LOCKED) — `get_diagnostics()` is SYNC; the Story 1-5 ASYNC scaffold is RETIRED end-to-end.** Architecture §6.2 (lines 678-737) lists `get_diagnostics() -> Dictionary` under "State introspection (synchronous, ≤1 ms)" alongside `is_supported() / is_connected() / get_public_key() / ...`. The Story 1-5 ASYNC stub was a placeholder that drifted from the spec — no architecture amendment needed; the spec has always said SYNC.
  - **Retirement scope (T2 + T3):**
    1. Kotlin: DELETE `mwaGetDiagnosticsFromJni(reqId: String)` from `GDExtensionAndroidPlugin.kt:306-309` (returns `emitNotImplemented`); ADD `@JvmStatic @UsedByGodot fun mwaQueryDiagnosticsFromJni(): String` mirroring `mwaQuerySessionStateFromJni` at line 325-333.
    2. C++ JNI: DELETE `MwaAndroidBridgeJni::get_diagnostics(request_id: String)` at `mwa_android_bridge_jni.cpp:533`; DELETE `g_mid_mwa_get_diagnostics_from_jni` jmethodID + cache-load line; ADD `MwaAndroidBridgeJni::query_diagnostics_json()` mirroring `query_session_state` at `:619`.
    3. C++ class: REPLACE `MobileWalletAdapter::get_diagnostics()` body at `src/mwa/mobile_wallet_adapter.cpp:258-260` (currently `return godot::Dictionary();`) with a sync call to `bridge->query_diagnostics_json()` + JSON parse → Dictionary. Binding at line 137 unchanged (`ClassDB::bind_method(D_METHOD("get_diagnostics"), ...)` is method-name-only — no signature change visible to GDScript).
  - **R8 keep:** A-10's `@UsedByGodot` blanket rule covers the new Kotlin entry. NO new keep rule needed.
  - **Rejected alternative — Option B keep async + add sync:** preserves the unused async scaffold + jmethodID + 1-arg path; adds Kotlin/C++ surface that nothing calls; doubles the maintenance footprint. Picked Option A (full retire).

- **DD-5-2-2 (LOCKED) — Correlation-trace recorder wiring: caller-side `recordOnEmit(...)` helper invoked alongside every native post; NO bridge-level wrapping.** Rationale:
  1. **9 emit sites** must record: 7 op-method terminal-signal sites (mwaConnect happy + error; mwaSignMessages; mwaSignTransactions; mwaSignAndSendTransactions; mwaReauthorize; mwaDisconnect; mwaDeauthorize) + 2 cancel-loop sites (mwaForgetAll cancel-loop at GDExtensionAndroidPlugin.kt:1408-1421; MwaLifecycleObserver.onDestroy cancel-loop from Story 5-3).
  2. **Helper signature:** `private fun recordOnEmit(requestId: String, sourceMethod: String, terminalSignal: String, startedAtMs: Long) { val now = clock(); diagnostics.recordCorrelationTrace(CorrelationTraceEntry(requestId, sourceMethod, terminalSignal, elapsedMs = now - startedAtMs, timestampMs = now)) }` — defined in `GDExtensionAndroidPlugin.kt` companion or as a method on the class; alternative path via NativeBridge wrapper considered + REJECTED (NativeBridge is currently a thin pass-through; widening it to know about diagnostics + InflightMap couples concerns).
  3. **Caller discipline:** at every `nativeBridge.postXxxCompleted(reqId, ...)` / `postMwaError(...)` / `postMwaTimeout(...)` / `postMwaCancelledLifecycle(...)` / `postReauthRequired(...)` site, the line BEFORE the post calls `recordOnEmit(reqId, sourceMethod, "xxx_completed" | "mwa_error" | "mwa_timeout" | "mwa_cancelled_lifecycle" | "reauth_required", startedAtMs)`. The `startedAtMs` value is sourced from the same site that calls `inflightMap.register(reqId, clock(), sourceMethod)` (per Story 4-2 3-arg register signature) — captured into a local `val startedAtMs = clock()` at op-method entry and threaded to the emit site OR re-fetched via `inflightMap.snapshot()[reqId]?.startedAtMs` if still in-flight (cleaner for cancel-loops).
  4. **`terminal_signal` enum (5-key entry's third value):** fixed vocabulary `{"connect_completed", "sign_messages_completed", "sign_transactions_completed", "sign_and_send_completed", "reauthorize_completed", "disconnect_completed", "deauthorize_completed", "mwa_error", "mwa_timeout", "mwa_cancelled_lifecycle", "reauth_required"}` — 11 values. T1 unit test asserts a typed sentinel for each op's happy + sad paths.
  - **Rejected alternative — bridge-level wrapping:** make NativeBridge own diagnostics + InflightMap; every post() method records before posting. Couples NativeBridge to two unrelated systems; tests must mock 2 extra collaborators per case; rejected.
  - **Rejected alternative — Aspect/AOP:** out of Kotlin's idiom; rejected.

- **DD-5-2-3 (LOCKED) — Non-Android `get_diagnostics()` returns 12-key Dictionary with empty-string / 0-int / empty-Array values; non-Android `get_device_posture()` returns 4-key all-false Dictionary.** Rationale: GDScript callers must NOT branch on `is_supported()` before calling diagnostics — the support runbook needs a stable shape regardless of platform. The C++ side handles non-Android via the existing `MobileWalletAdapter::is_supported()` early-return pattern: if `!is_supported()` return a hard-coded 12-key empty Dictionary (sdk_version still populated from a compile-time const; godot_version from `Engine::get_singleton()->get_version_info()`; everything else empty/zero/array-empty). `get_device_posture()` similarly returns `{rooted: false, debuggable: false, developer_options_on: false, adb_enabled: false}` on non-Android.
  - **Rejected alternative — return empty Dict:** breaks AC-1's "EXACTLY these top-level keys" — non-Android caller would get `{}` and a key-shape assertion would fail. Rejected.

- **DD-5-2-4 (LOCKED) — AC-5 reconciliation: `_SUGGESTED_WALLETS` exposes 4-key SUPERSET `{name, package_id, play_store_url, website_url}`.** Rationale: AC-5's literal 3-key requirement (`{name, package_id, play_store_url}`) is satisfied because the superset CONTAINS those 3 keys. Existing `website_url` retained for backward compat. `package_id` parsed from `play_store_url`'s `?id=` query param at const-init time (Phantom: `?id=app.phantom` → `app.phantom`; Solflare: `?id=com.solflare.mobile` → `com.solflare.mobile`; Backpack: `?id=app.backpack` → `app.backpack`). Implementation choice: hardcode the 3 package IDs in the const directly (cheaper than runtime URL parsing; the play store URLs are already hardcoded too).
  - **Rejected alternative — strict 3-key match (drop `website_url`):** breaks any existing caller that reads `website_url`; would require a coordinated migration. Rejected.
  - **Rejected alternative — keep existing 3-key shape (`website_url` instead of `package_id`):** spec compliance gap with AC-5; would require an amendment narrowing AC-5. Rejected.

- **DD-5-2-5 (LOCKED) — CR-3-3-A closure scope: `late_result_count` SURFACED via AC-1 12-key Dictionary; `cleanup_failed_count` NOT surfaced this story.** AC-1 spec text enumerates EXACTLY 12 keys; `cleanup_failed_count` is NOT among them. Story 5-2 surfaces what the spec requires; closure of CR-3-3-A's literal scope (lateResultCount visible) is achieved. `cleanup_failed_count` is a NEW concern (CR-5-2-A-pending — file at T6 if `cleanupFailedCount > 0` ever surfaces in support tickets); NOT a Story 5-2 deliverable.
  - **Rejected alternative — extend AC-1 to 13 keys including `cleanup_failed_count`:** changes the spec text without amendment; rejected. If a future support ticket needs the surface, file an amendment + a follow-up story.

### Concurrency Notes

- `recordCorrelationTrace` may be called from `Dispatchers.Default` (op-method coroutines) AND main thread (lifecycle observer onDestroy runs on main). Use `synchronized(correlationLock) { ... }` for both insert and snapshot. Alternative `ConcurrentLinkedDeque + AtomicInteger size` rejected — size cap requires read-modify-write atomicity that's not naturally expressed without locks.
- `mwaQueryDiagnosticsFromJni()` is invoked from the JNI bridge on whatever thread Godot calls it from (typically main). Reading `diagnostics.lastNCorrelationTrace` returns a defensive copy under lock — safe.
